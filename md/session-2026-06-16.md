## Changes Made

### 1. Fixed CSS & JS paths on all sub-pages
All pages inside `discovery/`, `info/`, `account/`, `purchase/`, and `error/` were using `../../css/` and `../../js/` (two levels up), landing outside the site root. Changed to `../css/` and `../js/`.

**Files affected:** All 18 sub-page HTML files.

### 2. Fixed CSS & JS paths on root-level pages
`index.html` and `interstitial.html` were using `../css/` (one level up, outside site root). Changed to `css/` and `js/` (no prefix — same directory).

### 5. Fixed logo in navigation (`shared.js`)
`assetBase` was set to `'../' + base`, adding an extra `../` that broke the logo path on all pages. Changed to `const assetBase = base;`.

---

### 6. Fixed nav "all active" bug on homepage (`shared.js`)
When the site is served from the root (e.g. via VESEL), `window.location.pathname.split('/').pop()` returns `''`. Since `anyString.endsWith('')` is always `true`, every nav link was marked active.

Fix: wrapped the active-state logic in `if (currentPage) { ... }` to skip when no page filename is present.
